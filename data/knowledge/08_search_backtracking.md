# 搜索与回溯题测试策略

## 常见测试策略

- 构造最小搜索空间，验证状态转移和答案格式。
- 构造无解、有唯一解、多解的数据。
- 对 BFS，覆盖起点即终点、目标不可达、需要绕障碍。
- 对 DFS 回溯，构造需要撤销选择、重复元素去重、剪枝边界的数据。
- 对双向搜索，构造起点和终点状态相同、状态空间很小、状态空间接近上限。
- 对 meet-in-the-middle，构造左右两半规模不均和答案跨两半组合的情况。

## 边界情况

- 搜索深度为 0 或 1。
- 图或网格没有可走边。
- 所有选择都合法，或所有选择都非法。
- 状态中包含多个维度，例如位置、钥匙、时间、方向。
- 递归深度可能超过语言默认栈限制。
- 剪枝条件刚好等于当前最优值。

## 容易卡掉错误代码的反例类型

- 回溯没有撤销标记或撤销顺序错误。
- BFS 标记访问时机错误，重复入队导致爆内存。
- DFS 没有记忆化，在重叠状态下指数爆炸。
- 剪枝过强，误删最优解。
- 状态哈希不完整，两个不同状态被当作同一个。
- 多源 BFS 忘记把所有起点初始入队。
- 双向 BFS 没有在较小 frontier 扩展，导致超时。
