# 图论题测试策略

## 常见测试策略

- 按图结构分层构造：空图、链、环、星形、树、DAG、稠密图、多个连通块。
- 对最短路题，分别构造无权图、正权图、边权差异极大、存在多条最短路的数据。
- 对可达性题，构造起点不可达目标、目标可达但需要绕路、多个入口同时可达。
- 对最小生成树题，构造图不连通、边权相同、存在多棵 MST、负权边。
- 对拓扑排序题，构造唯一拓扑序、多种拓扑序、有环、孤立点。
- 对状态图题，组合位置状态和附加状态，例如钥匙、已访问集合、能量、等级。
- 对动态开放节点或边的题，构造“先获得能力再回头访问”的路径。

## 边界情况

- `n=1,m=0`，起点就是终点。
- `m=0` 且目标不存在或不可达。
- 多条重边连接同一对点，权值不同。
- 边权为 `1`、最大边权、路径长度超过 `int`。
- 图中有自环，若题面允许；若不允许，可用于鲁棒性检查生成器不要误造。
- 起点或终点所在连通块很小，其他节点构成大连通块。
- 所有边权相同，最短路退化为 BFS 层数。
- DAG 中存在多个入度为 0 的点，或所有点链式依赖。

## 容易卡掉错误代码的反例类型

- Dijkstra 使用普通队列或未处理过期堆元素，导致复杂度或答案错误。
- `dist` 使用 `int`，长路径溢出为负数。
- 对重边只保留最后一条，而不是最小权边。
- 无向图只加了一条边。
- BFS 层数和边权混用，错误地用 BFS 解正权最短路。
- 拓扑排序没有检查最终弹出点数，无法识别有环。
- 并查集未路径压缩或合并方向错误，在大数据下超时。
- 状态最短路只记录点的最短距离，忘记把关键状态纳入 `dist[node][state]`。
