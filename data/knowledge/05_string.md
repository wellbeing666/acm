# 字符串题测试策略

## 常见测试策略

- 覆盖空串、单字符、所有字符相同、完全不同、周期串。
- 对模式匹配题，构造前缀和后缀高度重合的文本与模式。
- 对哈希题，构造大量重复子串，并使用双哈希或校验避免碰撞风险。
- 对 Trie 题，构造一个单词是另一个单词前缀的情况。
- 对回文题，覆盖奇数长度、偶数长度、全相同字符、无回文扩展。
- 对字典序题，构造大小写、数字、前缀相同但长度不同的字符串。
- 对多字符串题，构造重复字符串、空结果、所有字符串完全相同。

## 边界情况

- 字符串长度为 0 或 1，若题面允许。
- 模式串比文本串长。
- 匹配出现在开头、结尾、多个重叠位置。
- 字符集大小：只含小写、含大写、含数字或符号。
- 输入中可能包含空格时，不能使用普通 `cin >> s` 或 `split`。
- Unicode 通常不出现在 ACM 传统题，但若题面允许，需要按字符还是字节处理。

## 容易卡掉错误代码的反例类型

- KMP 的 `next` 或 `pi` 回退边界错误，无法处理重叠匹配。
- 字符串哈希忘记统一下标或幂次，子串哈希错位。
- 单哈希被碰撞或取模溢出影响。
- Trie 查询时未区分“路径存在”和“单词结束”。
- Manacher 半径和原串位置映射错误。
- 字典序比较误把短前缀排在后面。
- 多组数据中 Trie、计数数组、哈希数组未清空。
