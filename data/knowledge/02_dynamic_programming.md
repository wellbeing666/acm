# 动态规划题测试策略

## 常见测试策略

- 构造最小状态空间，人工枚举所有可能答案。
- 构造转移边界刚好成立和刚好不成立的样例。
- 对背包题，覆盖 0/1 背包、完全背包、多重背包中物品容量等于上限的情况。
- 对区间 DP，构造长度为 1、2、3 的区间，再构造需要跨区间合并的样例。
- 对树形 DP，构造单点树、链、星形树、完全二叉树。
- 对状态压缩 DP，构造 `k=0`、`k=1`、`k` 达到上限、状态全开和全关。
- 对计数 DP，构造答案为 0、1、很大需要取模的数据。

## 边界情况

- 初始状态是否需要设置为 0、负无穷或正无穷。
- 转移中第一个元素、最后一个元素、空前缀、空后缀。
- 容量为 0，物品价值为 0，重量为 0，若题面允许。
- 多个选择价值相同，但路径或方案数不同。
- `n` 很小但答案特殊，例如无法转移或只能选一个。
- `n` 最大时状态数量接近内存上限。
- 需要滚动数组时，循环方向影响答案。

## 容易卡掉错误代码的反例类型

- 0/1 背包把容量正序枚举，导致同一物品被重复选。
- 完全背包把容量倒序枚举，导致少算。
- 忘记初始化不可达状态，用默认 0 参与最大值转移。
- 区间 DP 枚举长度、左端点、分割点的边界少一格。
- 树形 DP 没有排除父节点，递归成环。
- 状态压缩中位运算优先级错误，例如 `mask & 1 << i == 0`。
- 计数 DP 忘记取模或减法后未加模数。
- 优化 DP 时单调队列、斜率优化的相等情况处理错误。
