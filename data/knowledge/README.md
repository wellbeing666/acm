# RAG 知识库索引

## 常见测试策略

本目录用于 ACM 测试用例生成 RAG。建议检索时优先使用题面解析出的 `problem_type` 和 `algorithm_tags`，再召回对应专题文件。通用文件适合作为每次生成测试用例时的基础上下文，专题文件适合作为补充上下文。

推荐召回顺序：

- `00_general_testing.md`
- 与 `problem_type` 匹配的专题文件
- 与 `algorithm_tags` 匹配的专题文件
- 若题目混合多种算法，召回多个专题并让模型合并策略

## 边界情况

知识库文件统一包含三个固定小节：

- 常见测试策略
- 边界情况
- 容易卡掉错误代码的反例类型

切分 Markdown 时建议按二级标题切块，同时保留 front matter 元数据，方便按题型过滤。

## 容易卡掉错误代码的反例类型

RAG 生成测试时不要只取随机数据。应要求模型至少生成：

- 最小规模测试
- 最大规模或压力测试
- 无解测试
- 多解或多最优解测试
- 针对错误贪心、错误边界、整数溢出、未初始化、多组数据污染的反例
