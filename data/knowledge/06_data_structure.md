# 数据结构题测试策略

## 常见测试策略

- 对区间查询题，覆盖单点查询、整段查询、左端点、右端点、空交集。
- 对线段树懒标记，构造多次覆盖、多次加法、覆盖后加法、加法后覆盖。
- 对树状数组，构造下标为 1、最大下标、重复更新同一位置。
- 对并查集，构造链式合并、重复合并、查询未连通元素。
- 对堆，构造插入、删除、取最值交替出现，以及大量相等元素。
- 对单调栈/队列，构造严格递增、严格递减、全相等序列。
- 对离散化，构造负数、大数、重复值、查询值不在原数组中。

## 边界情况

- `l=r` 的区间操作。
- 区间长度为 0 的非法输入应避免生成，除非题面允许。
- 更新值为 0、负数、最大值。
- 多组数据时内部数组、懒标记、父节点数组需要重置。
- 操作次数最大，所有操作集中在同一区间或同一点。
- 离散化后坐标数量远小于原值范围。

## 容易卡掉错误代码的反例类型

- 线段树懒标记下传顺序错误，覆盖和加法混用时答案错。
- 查询区间边界写错，`[l,r]` 和 `[l,r)` 混淆。
- 树状数组使用 0 下标导致死循环。
- 并查集未压缩路径，在链式数据下超时。
- 优先队列延迟删除时没有循环弹出失效元素。
- 单调队列窗口过期判断少一位。
- 离散化后忘记保留区间端点之间的空隙，影响覆盖长度类题。
